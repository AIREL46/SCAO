Adaptation du code de la e-p-433-v2 pour la e-p-wifi-v2

Problème sur la e-p-wifi-v2 (Résolu)

Quand le MKR WiFi est branché à l’ordinateur, celui-ci ne le reconnais pas, le port n’est pas disponible dans Arduino IDE. L’hypothèse étant que le MKR WiFi 1010 manque de puissance pour se connecter normalement à l’ordinateur. Cela est du au branchement de la breadboard (court-circuit ou composant H.S).

Pour vérifier l’hypothèse, touts les câbles branchés aux pins du MKR WiFi 1010 on été débranchés puis rebranchés un à un pour trouver sur quel pin le problèmes apparaît. Après l’essais de touts les pins, il se trouve que le pin fournissant le 3.3V (VCC) est le pin qui pose problème.

Pour vérifier ce qui pose défaut avec le 3.3V , on vient déconnecter un à un les composants branchés sur la breadboard utilisant le 3.3V (Emetteur 433MHz, condensateur, thermomètres digitaux). Après avoir testés touts les composant il se trouve que l’un des thermomètres digitaux est défectueux. Il a donc été retiré afin d’être remplacer.
