Version dégradé de mon projet sur Github

Suite à l’installation de Ubuntu 24.04 LTS, suivi du clonage de mon projet archivé sur la plateforme Github et d’une erreur de ma part lors de ce clonage, je me retrouve avec une version dégradé de mon projet. Par contre, en remontant 2 commits en arrière, j’en retrouve la version intégrale.

Les 2 numéros de commits successifs en date du 30/06/2025 qui ont induit cette dégradation sont : * 1 : 4095c99 * 2 : ef8fcc6

J’ai contacté Clément Ortiz qui m’a dépanné en me proposant la procédure suivante qui commence par l’installation de git-lfs que j’avais ommis de faire : * sudo apt update * sudo apt install git-lfs * mkdir SCAO_restore * cd SCAO_restore * git clone https://github.com/AIREL46/SCAO * cd SCAO * git revert ef8fcc6 –no-commit * git commit -m “Revert suppression” * git push origin master Les deux e-mails échangés avec Clément Ortiz sont : * 1-Version dégradé de mon projet sur Github * 2-Version dégradé de mon projet sur Github ## Restauration déjà-dup Voir la documentation Ubuntu
